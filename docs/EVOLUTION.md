# Evolution Log — Imeritas.Agent.DadJokes

Chronological record of significant decisions and changes.

---

## 2026-03-04 — Full implementation completed

- **Joke Collection Service**: Created `Joke` record model, `JokeData` static collection (20 jokes across 8 categories: tech, food, animals, work, science, sports, music, general), and `JokeCollectionService` with category lookup, random selection, and fallback logic.
- **DadJokesPlugin**: Singleton `IAgentPlugin` with `tell_joke` `[KernelFunction]`, `IClassificationContributor` (`/joke` slash command, classification examples/hints), and `IConfigurablePlugin<DadJokesPluginSettings>` with `MaxJokesPerSession` setting.
- **DadJokeOrchestrator**: Scoped `ITaskOrchestrator` for `dad_joke` task type (Priority 50) with category extraction from `inputData`, task lifecycle management, and `SaveTaskAsync` persistence.
- **Unit Tests**: 46 xUnit tests covering all public API surface — `JokeCollectionServiceTests` (13), `DadJokesPluginTests` (12), `DadJokeOrchestratorTests` (13 + 4 theory cases). NSubstitute used for `IStorageService` and `ILogger` mocking.
- **Key design decisions**:
  - Used `using AgentTaskStatus = Imeritas.Agent.Models.TaskStatus;` alias to avoid namespace conflict with `Imeritas.Agent.DadJokes.Models`
  - Two constructors on plugin: `PluginHostContext` for framework loader, `internal ILogger<T>` for unit testing (via `InternalsVisibleTo`)
  - `JokeCollectionService` is a plain C# class (not registered in DI) — instantiated directly by both plugin and orchestrator
  - `SlashCommandMapping.ScheduleFrequency` set to empty string (non-scheduled command)
